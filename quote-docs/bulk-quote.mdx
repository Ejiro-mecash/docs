The **meCash Bulk Quote API** lets you generate quotes for multiple transfers in a single request. Instead of creating individual quotes one by one, you submit all transfer amounts at once and receive a separate `quoteId` for each â€” ready to use with the [Bulk Payout API](/payout-docs/bulk-payout).

<Tip>Each item in the bulk quote request receives its own independent `quoteId` with its own pricing, fees, and expiry. Quotes expire after 10 minutes.</Tip>

## When to use bulk quotes

Use bulk quotes when you need to disburse funds to multiple beneficiaries and want to lock in pricing for all transfers upfront. Common scenarios include:
- **Payroll**: Quote salary amounts for all employees before disbursement.
- **Vendor payments**: Get pricing for multiple supplier payments at once.
- **Mass disbursements**: Quote commissions, rewards, or refunds in batch.

## How it works

<Steps>
### Prepare your transfer list

Gather all the amounts you need to send. Each item only requires a `sourceAmount` â€” the amount in your source currency.

### Create the bulk quote

Send all amounts in a single `POST /v1/bulk/quote` request along with the payment channel, source, and target corridor.

```bash cURL
curl --request POST 'https://{{baseURL}}/v1/bulk/quote' \
  --header 'Content-Type: application/json' \
  --header 'x-api-key: YOUR_API_KEY' \
  --data '{
    "paymentChannel": "BANK_TRANSFER",
    "items": [
        { "sourceAmount": 100.00 },
        { "sourceAmount": 250.50 }
    ],
    "source": { "currency": "NGN", "country": "NG" },
    "target": { "currency": "NGN", "country": "NG" }
}'
```

### Collect the quote IDs

The response returns an array of quotes, each with its own `quoteId`, rate, fee, and expiry. Store these IDs for the next step.

### Verify quotes (optional)

Fetch one or more quotes by ID to confirm pricing before executing payouts.

```bash cURL
curl --request GET 'https://{{baseURL}}/v1/bulk/quote?quoteIds=fa01c1ad-6c0f-4074-8221-ffc8fdaa16db&quoteIds=6809b2ab-1d4d-4436-8218-46667419eee7' \
  --header 'Content-Type: application/json' \
  --header 'x-api-key: YOUR_API_KEY'
```

### Use in bulk payout

Pass each `quoteId` into the corresponding order in your [`POST /v1/bulk/payout`](/payout-docs/bulk-payout) request.
</Steps>

<Tip>Only one Payment Channel and Route (Source and Target Currency/Country) is allowed per bulk quote request.</Tip>

## Request fields

| Field | Type | Description | Required |
|-------|------|-------------|----------|
| `paymentChannel` | String | Channel for the transfers (`BANK_TRANSFER`, `MOBILE_MONEY`, etc.). | âś… Yes |
| `items` | Array | List of objects, each with a `sourceAmount`. | âś… Yes |
| `items[].sourceAmount` | Number | Amount to send in the source currency. | âś… Yes |
| `source.currency` | String | Source currency (ISO 4217). | âś… Yes |
| `source.country` | String | Source country (ISO 3166-1 alpha-2). | âś… Yes |
| `target.currency` | String | Target currency (ISO 4217). | âś… Yes |
| `target.country` | String | Target country (ISO 3166-1 alpha-2). | âś… Yes |

## Response fields

Each array item contains a `quote` object with the standard quote structure:

| Field | Type | Description |
|-------|------|-------------|
| `quote.id` | String (UUID) | Unique identifier for this quote. Use when creating the bulk payout. |
| `quote.source.amount` | Number | Amount in the source currency. |
| `quote.target.amount` | Number | Amount the recipient will receive. |
| `quote.rate` | Number | Exchange rate applied. |
| `quote.fee.amount` | Number | Fee charged for this transfer. |
| `quote.source.currency` | String | Source currency code. |
| `quote.target.currency` | String | Target currency code. |
| `quote.expiresInSeconds` | Integer | Seconds until this quote expires (typically 600). |
