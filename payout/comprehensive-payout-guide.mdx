This guide covers all payout scenarios supported by the meCash API, including different currencies, payment channels, and recipient types.

## Overview

The Payout API allows you to send money to recipients worldwide using various payment methods. All payouts require a valid quote ID from the Quote API.

## Endpoint

**`POST`** `{{baseURL}}/v2/payout`

### Authentication

All requests require an API key in the header:

```bash
x-api-key: YOUR_API_KEY
```

## Request Format

All payout requests use `multipart/form-data` with a `transaction` field containing JSON data.

## Payout Examples by Currency

### 1. Nigerian Naira (NGN) Payout

<CodeGroup>
```bash cURL
curl --location '{{baseURL}}/v2/payout' \
--header 'x-api-key: YOUR_API_KEY' \
--form 'transaction={
  "recipient": {
    "name": "John Doe",
    "paymentChannel": "BANK_TRANSFER",
    "currency": "NGN",
    "country": "NG",
    "type": "INDIVIDUAL",
    "account": {
      "bankName": "Guaranty Trust Bank",
      "sortCode": "058",
      "accountNumber": "0010401234"
    }
  },
  "quoteId": "your-quote-id-here",
  "reason": "FAMILY_SUPPORT",
  "remark": "Monthly support"
}'
```

```javascript JavaScript
const formData = new FormData();
const transaction = {
  recipient: {
    name: "John Doe",
    paymentChannel: "BANK_TRANSFER",
    currency: "NGN",
    country: "NG",
    type: "INDIVIDUAL",
    account: {
      bankName: "Guaranty Trust Bank",
      sortCode: "058",
      accountNumber: "0010401234"
    }
  },
  quoteId: "your-quote-id-here",
  reason: "FAMILY_SUPPORT",
  remark: "Monthly support"
};

formData.append('transaction', JSON.stringify(transaction));

fetch('{{baseURL}}/v2/payout', {
  method: 'POST',
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  },
  body: formData
})
.then(response => response.json())
.then(data => console.log(data));
```

```python Python

import json

url = "{{baseURL}}/v2/payout"

    "x-api-key": "YOUR_API_KEY"
}

        "country": "NG",
        "type": "INDIVIDUAL",
        "account": {
            "bankName": "Guaranty Trust Bank",
            "sortCode": "058",
            "accountNumber": "0010401234"
        }
    },
    "quoteId": "your-quote-id-here",
    "reason": "FAMILY_SUPPORT",
    "remark": "Monthly support"
}

    'transaction': (None, json.dumps(transaction))
```

## Error Handling

### Common Error Responses

| **Status** | **Message** | **Cause** | **How to fix** |
|--------|---------|-------|------------|
| 400 | `Insufficient wallet balance for transaction` | Your wallet lacks the funds required to cover the payout. | Fund the wallet or lower the payout amount before retrying. |
| 400 | `Quotes only last for 10 minutes` | The quote expired before you attempted the payout. | Request a fresh quote and submit the payout immediately. |
| 400 | `Invalid country or currency ISO code` | Unsupported country/currency combination in `recipient` or `quote`. | Pass valid ISO codes for corridors enabled on your workspace. |
| 400 | `Account name mismatch, use account enquiry endpoint` | Provided account name differs from the latest bank enquiry response. | Re-run account enquiry and align your payload with the returned details. |
| 400 | `Invalid Bank Sortcode` | Sort code is missing, malformed, or not supported for the corridor. | Retrieve the sort code from the bank list API and validate the format. |
| 400 | `{field_name} cannot be empty` | Required recipient field (name, account number, etc.) missing in payload. | Populate every mandatory field before submitting the request. |
| 401 | `API key missing or incorrect` | `x-api-key` header absent, invalid, or from the wrong environment. | Include the correct API key for sandbox or production. |
| 403 | `IP not whitelisted` | Request originated from a non-whitelisted IP address. | Add your server IP in the dashboard allowlist. |
| 404 | `The requested endpoint does not exist` | Incorrect URL path or API version. | Confirm you are calling `POST /v2/payout` on the right environment host. |
| 422 | `Invalid quote ID provided` | Quote ID is missing, invalid, expired, or already consumed. | Create a new quote and reference its `id` when resubmitting. |
| 500 | `Service temporarily unavailable` | Temporary backend/service disruption. | Retry with exponential backoff and contact support if persistent. |

### Error Response Format

```json
{
  "status": "error",
  "errorCode": "ERROR_CODE",
  "message": "Human readable error message"
}
```

## Best Practices

### Before Creating Payouts

1. **Generate a Quote First**: Always create a quote using the Quote API before creating a payout
2. **Validate Recipient Data**: Ensure all recipient information is accurate and complete
3. **Check Rate Limits**: Be aware of API rate limits (100 requests/minute)

### Security

1. **Use HTTPS**: Always use HTTPS endpoints
2. **Secure API Keys**: Store API keys securely, never in client-side code
3. **IP Whitelisting**: Ensure your server IP is whitelisted

### Monitoring

1. **Track Transaction States**: Monitor transaction states via webhooks
2. **Handle Failures**: Implement retry logic for failed transactions
3. **Log Everything**: Log all API calls for debugging and auditing
