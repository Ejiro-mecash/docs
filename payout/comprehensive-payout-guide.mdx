## Comprehensive Payout Guide

This guide covers all payout scenarios supported by the meCash API, including different currencies, payment channels, and recipient types.

## Overview

The Payout API allows you to send money to recipients worldwide using various payment methods. All payouts require a valid quote ID from the Quote API.

## Endpoint

**`POST`** `{{baseURL}}/v2/payout`

### Authentication

All requests require an API key in the header:

```bash
x-api-key: YOUR_API_KEY
```

## Request Format

All payout requests use `multipart/form-data` with a `transaction` field containing JSON data.

## Payout Examples by Currency

### 1. Nigerian Naira (NGN) Payout

<CodeGroup>

```bash cURL
curl --location '{{baseURL}}/v2/payout' \
--header 'x-api-key: YOUR_API_KEY' \
--form 'transaction={
  "recipient": {
    "name": "John Doe",
    "paymentChannel": "BANK_TRANSFER",
    "currency": "NGN",
    "country": "NG",
    "type": "INDIVIDUAL",
    "account": {
      "bankName": "Guaranty Trust Bank",
      "sortCode": "058",
      "accountNumber": "0010401234"
    }
  },
  "quoteId": "your-quote-id-here",
  "reason": "FAMILY_SUPPORT",
  "remark": "Monthly support"
}'
```

```javascript JavaScript
const formData = new FormData();
const transaction = {
  recipient: {
    name: "John Doe",
    paymentChannel: "BANK_TRANSFER",
    currency: "NGN",
    country: "NG",
    type: "INDIVIDUAL",
    account: {
      bankName: "Guaranty Trust Bank",
      sortCode: "058",
      accountNumber: "0010401234"
    }
  },
  quoteId: "your-quote-id-here",
  reason: "FAMILY_SUPPORT",
  remark: "Monthly support"
};

formData.append('transaction', JSON.stringify(transaction));

fetch('{{baseURL}}/v2/payout', {
  method: 'POST',
  headers: {
    'x-api-key': 'YOUR_API_KEY'
  },
  body: formData
})
.then(response => response.json())
.then(data => console.log(data));
```

```python Python
import requests
import json

url = "{{baseURL}}/v2/payout"
headers = {
    "x-api-key": "YOUR_API_KEY"
}

transaction = {
    "recipient": {
        "name": "John Doe",
        "paymentChannel": "BANK_TRANSFER",
        "currency": "NGN",
        "country": "NG",
        "type": "INDIVIDUAL",
        "account": {
            "bankName": "Guaranty Trust Bank",
            "sortCode": "058",
            "accountNumber": "0010401234"
        }
    },
    "quoteId": "your-quote-id-here",
    "reason": "FAMILY_SUPPORT",
    "remark": "Monthly support"
}

files = {
    'transaction': (None, json.dumps(transaction))
}

response = requests.post(url, headers=headers, files=files)
print(response.json())
```

## Request Parameters Reference

### Transaction Object

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `recipient` | Object | ‚úÖ | Recipient details (structure varies by currency) |
| `quoteId` | String | ‚úÖ | Valid quote ID from Quote API |
| `reason` | String | ‚úÖ | Transaction reason (see enum values below) |
| `remark` | String | ‚ùå | Optional transaction note |

### Reason Enum Values

- `FAMILY_SUPPORT` - Family financial support
- `GIFT` - Gift payments
- `SALARY_PAYMENT` - Salary payments
- `EDUCATIONAL_FEES` - Educational expenses
- `BUSINESS_INVOICE` - Business invoice payments
- `OTHER` - Other purposes

## Error Handling

### Common Error Responses

| Status Code | Error Type | Description | Solution |
|-------------|------------|-------------|----------|
| 400 | Bad Request | Invalid parameters | Check request payload |
| 401 | Unauthorized | Invalid API key | Verify API key |
| 403 | Forbidden | IP not whitelisted | Whitelist your IP |
| 422 | Unprocessable Entity | Invalid quote ID | Generate new quote |

### Error Response Format

```json
{
  "status": "error",
  "errorCode": "ERROR_CODE",
  "message": "Human readable error message"
}
```

## Best Practices

### Before Creating Payouts

1. **Generate a Quote First**: Always create a quote using the Quote API before creating a payout
2. **Validate Recipient Data**: Ensure all recipient information is accurate and complete
3. **Check Rate Limits**: Be aware of API rate limits (100 requests/minute)

### Security

1. **Use HTTPS**: Always use HTTPS endpoints
2. **Secure API Keys**: Store API keys securely, never in client-side code
3. **IP Whitelisting**: Ensure your server IP is whitelisted

### Monitoring

1. **Track Transaction States**: Monitor transaction states via webhooks
2. **Handle Failures**: Implement retry logic for failed transactions
3. **Log Everything**: Log all API calls for debugging and auditing
