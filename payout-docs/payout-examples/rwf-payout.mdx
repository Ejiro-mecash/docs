title: '🇷🇼 RWF Payout'
---

This endpoint allows you to process payouts in **Rwandan Franc (RWF)** to recipients in Rwanda using mobile money or bank transfers.

---

## Endpoint

**POST:** `https://sandboxapi.me-cash.com/v2/payout`

---

## Supported Payment Channels

RWF payouts support the following payment channel:

| **Channel** | **Description** | **Use Case** |
| :---------- | :-------------- | :----------- |
| `MOBILE_WALLET` | Mobile Money transfers (e.g., MTN MoMo) | Instant transfers to mobile wallets |


---

## Request Details

### Headers

Include these headers in your request:

| **Header** | **Value** | **Required** |
| :------------- | :----------------- | :------- |
| `Content-Type` | `application/json` | ✅ Yes   |
| `x-api-key`    | `YOUR_API_KEY`     | ✅ Yes   |

### Request Body Breakdown

| **Field** | **Type** | **Description** | **Required** |
| :---------------------------------- | :------ | :-------------------------------------------------------------------------- | :------- |
| `quoteId` | String | The unique ID of the quote for this payout. | ✅ Yes |
| `reason` | String | The purpose of the transfer. | ✅ Yes |
| `remark` | String | An optional, additional note about the transaction. | ❔ No |
| `recipient` | Object | An object containing all details about the beneficiary. | ✅ Yes |
| `recipient.name` | String | The full name of the recipient. | ✅ Yes |
| `recipient.account` | Object | An object containing the recipient's account details. | ✅ Yes |
| `recipient.account.accountNumber` | String | The recipient's mobile money number or bank account number. | ✅ Yes |
| `recipient.account.mobileProvider` | String | The mobile money provider code (e.g., `MTN_MOMO_RWA`). See [Get Mobile Networks](/collection/get-mobile-money-operators) for a list of supported codes. | ✅ Yes (for Mobile Wallet) |
| `recipient.paymentChannel` | String | The payment method. Enum: `MOBILE_WALLET`, `BANK_TRANSFER`. | ✅ Yes |
| `recipient.currency` | String | The ISO currency code. Must be `RWF`. | ✅ Yes |
| `recipient.country` | String | The recipient's two-letter ISO country code. Must be `RW`. | ✅ Yes |

<Tip>
To retrieve the correct `mobileProvider` codes for different countries and networks, use the [Get Mobile Networks](/collection/get-mobile-money-operators) endpoint.
</Tip>

## **Payout Transaction States**

The payout transaction can have one of the following states:

| **State** | **Description** |
|------------|-------------------------------------------|
| `COMPLETED` | The payout was successfully processed.   |
| `PENDING`   | The payout is still being processed.     |
| `FAILED`    | The payout failed to process.           |
| `REFUNDED`  | The payout has been sent back to sender. |

---

## Error Handling

| **Status Code** | **Meaning** | **Example Response** |
|:------------|:-----------------------|:------------------------------------------------|
| `400`       | Insufficient Balance   | Insufficient Balance                            |
| `401`       | Unauthorized           | Invalid API key provided.                       |
| `403`       | Forbidden              | IP not whitelisted                              |
| `404`       | Not Found              | The requested endpoint does not exist.          |
| `422`       | Unprocessable Entity   | Invalid quote ID provided.                      |
| `500`       | Internal Server Error  | An internal error occurred.                     |

---

## **Best Practices**
✅ Ensure `quoteId` is valid and linked to an existing quote.
✅ Confirm that the recipient's **mobile number** or **bank account number** is correct.
✅ Use a valid API key in the headers.
✅ Handle error responses correctly in your integration.

<Tip>
**Testing Failed Payouts:** In the sandbox environment, you can simulate a failed payout by setting `"remark": "fail"` in your request. This triggers a `payout.failed` webhook, allowing you to test your error handling.
</Tip>
