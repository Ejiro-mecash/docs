---
title: 'Get Wallets API'
---


Use this endpoint to fetch the wallet that corresponds to a currency and country pair. This is helpful when you need to select the right wallet before initiating payouts or checking balances.

## Endpoint

**GET** `{{baseURL}}/v1/wallet?currency={currency}&country={country}`

### Query parameters

| Parameter | Type   | Description | Required | Example |
|-----------|--------|-------------|----------|---------|
| `currency` | String | Three-letter ISO 4217 currency code for the wallet you want to retrieve. | ✅ Yes | `NGN` |
| `country` | String | Two-letter ISO 3166-1 alpha-2 country code tied to the wallet. | ✅ Yes | `NG` |

### Request headers

| Header | Type | Description | Required | Example |
|--------|------|-------------|----------|---------|
| `x-api-key` | String | Your meCash API key used for authentication. | ✅ Yes | `YOUR_API_KEY` |
| `Content-Type` | String | Indicates the media type of the request. | ✅ Yes | `application/json` |

## Successful response

```json copy
{
  "message": "wallets fetched successfully",
  "status": "success",
  "data": [
    {
      "id": "cfd2de64-a896-4405-8736-xxxxxxxxxxxxx",
      "balance": 23094071,
      "currency": "NGN",
      "country": "NG"
    }
  ]
}
```

### Field reference

| Field | Type | Description |
|-------|------|-------------|
| `message` | String | High-level description of the outcome. |
| `status` | String | `success` if the request succeeded, otherwise `error`. |
| `data` | Array\<Wallet\> | Array containing wallet objects that matched the filter. |

**Wallet object**

| Field | Type | Description |
|-------|------|-------------|
| `id` | String | Unique identifier for the wallet. |
| `balance` | Number | Available balance in the wallet's base currency. |
| `currency` | String | Currency code for the wallet balance. |
| `country` | String | Country code associated with the wallet. |

## Error responses

| Status | Meaning | Example message | Next steps |
|--------|---------|-----------------|------------|
| 400 | Missing or invalid query parameters. | `currency and country are required` | Confirm that both query parameters are present and use valid ISO codes. |
| 401 | Authentication failed. | `Invalid API key.` | Verify you are using the correct `x-api-key` and that it is still active. |
| 403 | IP address not whitelisted. | `IP not whitelisted.` | Add your IP address to the allowlist from the dashboard. |
| 404 | No wallet found for the supplied filters. | `Wallet not found.` | Create the wallet for that currency/country or double-check the filters. |
| 422 | Parameters are well-formed but invalid. | `Invalid parameter value.` | Make sure the currency and country codes are supported. |
| 429 | Rate limit exceeded. | `Too many requests.` | Add retry logic with exponential backoff. |
| 500 | Unexpected server error. | `Internal server error.` | Retry later or contact support if the issue persists. |

## Best practices

- Authenticate every request with a valid `x-api-key`.
- Cache wallet lookups locally when possible to reduce rate limit usage.
- Handle `404` responses gracefully by prompting users to create the missing wallet.
- Log response bodies for non-2xx results to speed up troubleshooting.
