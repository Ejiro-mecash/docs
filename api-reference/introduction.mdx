# API Reference

Welcome to the meCash API Reference! This comprehensive guide provides everything you need to integrate with the meCash platform, from authentication to advanced features.

## Getting Started

The meCash API allows you to:
- **Send and receive payments** across multiple currencies and countries
- **Manage virtual accounts** for seamless payment collection
- **Handle cryptocurrency operations** through our RAMP API
- **Track transactions** with real-time status updates
- **Create quotes** for currency conversions

## Authentication

All API requests require authentication using your API key:

```bash
curl -H "x-api-key: API_KEY:YOUR_API_KEY" \
     https://sandboxapi.me-cash.com/v1/wallet
```

<Note>
Your API key can be obtained from your **Dashboard → Developers → API Keys**. Keep it secure and never expose it in client-side code.
</Note>

## Interactive Playground

The API Reference includes an interactive playground where you can test endpoints directly in your browser:

<Steps>
  <Step title="Select Environment">
    Select the **Sandbox** environment.
  </Step>
  <Step title="Add Your API Key">
    Click the **Authorize** button and enter your API key in the format: `API_KEY:YOUR_API_KEY`
  </Step>
  <Step title="Try Endpoints">
    Click on any endpoint to expand it, then click **Try it out** to test the API directly.
  </Step>
  <Step title="View Responses">
    See real-time responses, status codes, and error messages for each request.
  </Step>
</Steps>

<Warning>
The playground uses your actual API credentials. Be cautious when testing with production endpoints and real data.
</Warning>

## Response Format

All API responses follow a consistent format:

```json
{
  "status": "success",
  "message": "Operation completed successfully",
  "data": {
    // Response data here
  }
}
```

### Error Responses

Error responses include detailed information:

```json
{
  "status": "error",
  "errorCode": "INVALID_PARAMETER",
  "message": "A request parameter is invalid."
}
```

## Webhooks

Stay updated with real-time notifications using webhooks:

<Accordion title="Supported Events">
- **Payout Events**: Payment status updates, completion notifications
- **Collection Events**: Incoming payment notifications
- **Virtual Account Events**: Account creation and payment received
- **Ramp Events**: Crypto transaction updates
</Accordion>

<Accordion title="Webhook Security">
- All webhooks are signed with HMAC-SHA256
- Verify webhook signatures to ensure authenticity
- Use HTTPS endpoints for webhook URLs
</Accordion>

## Quick Start Examples

<Accordion title="Create a Quote">
```bash
curl -X POST "https://sandboxapi.me-cash.com/v1/quote" \
  -H "x-api-key: API_KEY:YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "paymentChannel": "BANK_TRANSFER",
    "source": {
      "amount": 1000,
      "country": "NG",
      "currency": "NGN"
    },
    "target": {
      "country": "US",
      "currency": "USD"
    }
  }'
```
</Accordion>

<Accordion title="Create a Payout">
```bash
curl -X POST "https://sandboxapi.me-cash.com/v1/payout" \
  -H "x-api-key: API_KEY:YOUR_API_KEY" \
  -H "Content-Type: multipart/form-data" \
  -F 'transaction={
    "recipient": {
      "name": "John Doe",
      "account": {
        "bankName": "Access Bank",
        "sortCode": "044",
        "accountNumber": "1234567890"
      },
      "paymentChannel": "BANK_TRANSFER",
      "country": "NG",
      "stored": false
    },
    "quoteId": "your-quote-id",
    "reason": "Payment",
    "remark": ""
  }'
```
</Accordion>

<Accordion title="Get Transaction">
```bash
curl -X GET "https://sandboxapi.me-cash.com/v1/transaction/transaction-id" \
  -H "x-api-key: API_KEY:YOUR_API_KEY"
```
</Accordion>
